## 1. このプロジェクトについて

このプロジェクトは **MATTY（マッティー）** の正式開発プロジェクトです。

- **MATTY（マッティー）**：バー・スナック等の接客業向け、タブレット第一（tablet-first）の店舗CRM（顧客管理）PWAアプリ
- **最優先事項**：営業中30秒以内で来店登録が完了すること
- **開発方針**：シンプル・大きな文字・営業中の使いやすさを最優先する（「シンプルさ最優先／Simplicity-first UX」）

最新版ソースコードは

**`repomix-output.xml`**

を正とします。

新しい修正がGitHubへ反映されたら、**必ず `make-repomix.ps1` を実行して `repomix-output.xml` を再生成し、Project内の `repomix-output.xml` を最新版へ置き換えてください。**

過去チャットや古いコードではなく、

- GitHub最新版
- Project内の `repomix-output.xml`

を正として開発を行ってください。**過去チャットの内容やAI自身の記憶は、これらより優先しないでください。**

## 3. 開発ルール

修正時は以下を必ず守ってください。

- 既存仕様を壊さない
- 営業中の使いやすさを優先する
- 文字は大きく
- シンプルなUI
- 処理速度を優先する
- 不要な機能は追加しない
- 勝手に仕様変更しない

### 3.1 実装上の主要ルール（詳細は `03_AI_Guidelines.md` 参照）

- **完全なコードを出力する**：修正時は該当ファイルの完全な内容を出力する。差分の一部だけを渡さない（ユーザーがGitHub上のファイルを直接置き換える運用のため）。
- **推測で機能を変えない**：仕様が曖昧な場合は、複数の解釈がありうることを明記し、実装を進める前にユーザーへ確認する。仕様書・ユーザーの指示文を正とし、古いチャット内容やAI自身の記憶を優先しない。
- **DBスキーマは勝手に変更・削除しない**：新しいテーブル・カラムの追加、既存カラムの削除・型変更は、ユーザーが明示的に承認しない限り行わない。既存データを書き換えるSQLが必要な場合は、**実行せずSQL文だけを提示**し、実行はユーザー自身に委ねる。RLSポリシー・トリガーの変更は特に慎重に。
- 複数テーブルにまたがる多対多の登録・更新（同伴者・タグ等）は、既存のRPC（`create_visit_with_details`等）の流用・拡張を優先する。安易に複数テーブルへ直接UPDATEを積み重ねない。
- ユーザーはGitHub Web UIでファイルをアップロード・削除する形で反映する非エンジニアに近い運用者です。「どのファイルをどこに置くか」を具体的なパスで明記してください。フォルダ構成そのものを変える変更（旧フォルダの削除要否等）は特に明示してください。

## 10. 補足：現状把握のための参考情報

作業前に流し読みしておくと役立つ、現時点（`04_Development_History.md` 記載時点）の状況です。詳細は同ファイルを参照してください。

- 主要技術：Next.js（App Router）／React／TypeScript／Tailwind CSS v4／Supabase／Vercel自動デプロイ／PWA
- Next.js 16特有の注意点：ミドルウェアは `middleware.ts` ではなく `src/proxy.ts`（`proxy`関数をexport）。Server Actionsファイル（`"use server"`）はasync関数以外をexportできないため、型・定数は隣接する `xxx-state.ts` に分離する。
- ログイン後の全画面は `src/app/(app)/` 配下（共通レイアウトでヘッダー・ナビ・プロフィール取得を一元化）。新しいページも必ずこのグループ内に置く。
- PWAのService Worker（`public/sw.js`）は現在キルスイッチ状態（オフラインキャッシュ機能は無効化中）。オフライン検索・同期キューの基盤コード自体は存在するが、動作検証は未完了（TODO）。
- 既知の未解決事項・TODOは `04_Development_History.md` の「4. 既知のバグ・未解決事項」「5. 次にやること」を参照。
